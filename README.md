## 解答作成パイプライン（このリポの作り方）

解答例は、過去問の転記 tex を入力に、次の **検証ファースト** な手順で作成しています。
同じ手順で新しい年度・科目を追加できます。

```
過去問PDF
  └─(1) 転記 ─────────▶ 01_exams/<年度>/ … 問題文＋図(tikz) を tex 化
        └─(2) 読解・要約で誤読チェック
              └─(3) 検証ファースト ─▶ 02_solutions/<年度>/verify.py
                     （sympy/数値で全小問を検算、全 PASS になるまで本文を書かない）
                    └─(4) 執筆 ─────▶ doshisha_bme_<年度>_<科目>.tex
                           （前提整理 → 小問 Step 分割 → 最終解答は \ans{} で boxed・大問ごとに改ページ）
                          └─(5) コンパイル（XeLaTeX ×2、警告/はみ出し 0）
                                └─(6) 目視確認（PDFを画像化して1ページずつ）
                                      └─(7) クロスチェック（verify.py の値 ⇄ PDF の boxed 解答）
```

**並列化**: 年度＝1ジョブとして独立なので、年度ごとに並列実行できます（混入を避けるため科目は跨がない）。
本リポは各科目とも「まず 1 年度だけ仕上げて粒度を確定 → 残り年度を並列展開」という流れで作成しました。

**パイプラインの原則**
- **検証ファースト**: tex を書く前に `verify.py` を全 PASS（検算不能な図読み取り等は理由つき SKIP）にする。
- **粒度**: 行間を埋める（「明らか」で飛ばさない）。最終解答は必ず boxed、各大問に検算を最低 1 箇所、文体は常体。
- **正本はリポジトリ**: 状態は会話ではなく tex・コミット・分野マップに残す。

## ビルド方法

- コンパイラは **XeLaTeX 固定**（`pdflatex` 不可）。和文フォントは Noto Serif/Sans CJK JP（環境に応じて差し替え可）。
- 解答 1 本をビルド（年度フォルダ内で 2 回）:
  ```bash
  cd 04_材料力学/02_solutions/2012
  xelatex doshisha_bme_2012_mom.tex
  xelatex doshisha_bme_2012_mom.tex
  ```
  > 一部科目は tex を `02_solutions/<年度>/tex/` 下に置いています（その場合は `tex/` 内でビルド）。
- 検算の実行（依存: `sympy`, `mpmath`, `numpy` ほか）:
  ```bash
  cd 04_材料力学/02_solutions/2012
  python3 verify.py     # 各小問の [PASS]/[FAIL]/[SKIP] とサマリを表示
  ```
