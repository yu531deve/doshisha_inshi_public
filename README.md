# doshisha_inshi — 同志社 生命医科学研究科 医工学コース 院試 学習リポジトリ

同志社大学大学院 生命医科学研究科 **医工学コース**の院試過去問演習を、
**教師あり学習パイプライン**に見立てて運用する個人用リポジトリ。

## 理念（ML アナロジー）
- **データ** = 過去問（各科目 `01_exams/`）。
- **正解ラベル** = 自作の解答例（`02_solutions/`）。各解答は隣の `verify.py` で**機械検算**する。
- **学習器** = 人間（所有者）。型をモジュール単位で習得する。
- **データ拡張** = AI 生成の類似問題（`03_augmentation/`）。型の汎化を狙う（knowledge distillation 寄り）。
- **HELDOUT（test split）は設けない。** 全年度を作成対象にする。

## 構成（科目が最上位、各科目が独立フルセット）
4科目をそれぞれ独立した小プロジェクトとして扱う（別エージェント／別セッションで並行可、混入事故に強い）。

```
doshisha_inshi/
├ 01_数学/        ┐  各科目の中身は同一構成:
├ 02_力学/        │    00_common/  preamble.sty, verify_template.py（科目ごとに4本管理）
├ 03_制御工学/    │    01_exams/<年度>/    過去問PDF + 転記tex
├ 04_材料力学/    ┘    02_solutions/<年度>/ 解答tex/pdf + verify.py（★検算は解答の隣）
│                      03_augmentation/    AI生成の類似問題
│                      04_notes/           科目ノート
│                      05_meta/            分野マップ・学習ログ（転記後に作る）
│                      参考/               体裁の手本（東大版。体裁のみ）
│                      build/  original/
├ CLAUDE.md       リポジトリ共通の作業ガード（科目を跨ぐな）
└ README.md
```

## 想定ワークフロー
1. 過去問PDFを各科目の `original/` に置き、見やすく **TeX に転記**（`01_exams/<年度>/`）。
2. エージェントA: 転記 TeX → PDF へエクスポート。
3. エージェントB: 転記 TeX を元に**解答例**を作成（検証ファースト → tex → PDF）。
4. **初回は1年度のみ**回して粒度を確定してから横展開する。

## 重要原則（仕様書から引き継ぎ）
- **混入禁止**: 東大版は「体裁・粒度の手本」としてのみ使う。分野分類・問題内容・命名・出所を持ち込まない。
  医工学の分野分類は転記済み過去問から各科目で作る。
- **検証ファースト**: tex を書く前に `verify.py` で検算し、全 PASS（or 理由つき SKIP）にする。
- **目視確認**: コンパイル結果は必ず画像化して1ページずつ見る。
- **状態の正本はリポジトリ**: 会話でなく tex・コミット・分野マップに状態を残す。

## ビルド
- コンパイラは **XeLaTeX 固定**（`pdflatex` 不可）。和文は Noto Serif CJK JP（未導入なら Sans 等へ）。
- 解答1本のビルド（年度フォルダ内で2回）:
  ```bash
  cd 01_数学/02_solutions/2026
  xelatex doshisha_bme_2026_math.tex
  xelatex doshisha_bme_2026_math.tex
  ```
- 検算: `cd 01_数学/02_solutions/2026 && python3 verify.py`（依存: sympy, mpmath, numpy ほか）

## 著作権
過去問の著作権は同志社大学にある。本リポジトリは **private** で運用する。
