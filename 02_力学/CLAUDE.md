## 解答作成ワークフロー（仕様書§4、厳守）
1. **読解**: スキャンは `pdftoppm -png -r 150`（不鮮明なら `-r 300`）で画像化し全小問を読む。
   要約を出して誤読を自己点検。判読困難な箇所は推測で埋めず `rem`（注意）環境で明記。
2. **検証ファースト**: `00_common/verify_template.py` を `02_solutions/<年度>/verify.py` にコピーし、
   全小問を検算。**全 PASS（または SKIP に理由）になるまで tex を書き始めない。**
   - この科目の検算方針: 運動方程式の解・周期・固有角振動数・エネルギー保存・運動量などは SymPy/数値で検算できる。一方、自由体図のモデル化・図からの幾何読み取り・物理的近似の妥当性は数値検算の対象外なので、`note_unverifiable()` で理由を残す。最低でも各大問の最終量（数値）は1つは検算する。
3. **執筆**: `02_solutions/<年度>/doshisha_bme_<年度>_mechanics.tex` を作成。`00_common/preamble.sty` を使い、
   体裁・粒度は `参考/` のPDFに揃える（前提整理 → 小問 → Step分割 → 最終解答は `\ans{}` で boxed）。
4. **コンパイル**: 年度フォルダ内で `xelatex` を2回。警告・はみ出しゼロまで直す。
5. **目視確認**: 生成PDFを1ページずつ画像化し、数式崩れ・はみ出し・図の不備を確認。
6. **クロスチェック**: verify.py の値と PDF 中の boxed 解答が一致するか突き合わせ。

## 粒度（仕様書§6 / `参考/` と同一に）
- 行間を埋める（「明らか」「同様に」で飛ばさない）。最終解答は必ず boxed。
- 検算を本文に最低1箇所/大問（「検算:…✓」）。場合分け・境界・符号条件を明示。
- 文体は日本語の常体（「である」調）。

## 命名規則
- 解答 tex: `02_solutions/<年度>/doshisha_bme_<年度>_mechanics.tex`（年度=AY表記）
- 検算:     `02_solutions/<年度>/verify.py`（解答と同居。トップに集約しない）
- 類似問題: `03_augmentation/aug_<連番>_<短い英語名>.tex`

## 当面の進め方
- **初回は1年度のみ**仕上げて粒度を確定する（仕様書「段階的に刻む」）。

- HELDOUT は設けない（全年度が作成対象）。
